# Рефактор E2E тестов

## Контекст

Сейчас все e2e-тесты лежат в `tests/image-object2/`, хотя папка исторически относится к старому тестовому объекту `image-object2`.
Это вводит в заблуждение и не отражает фактическое покрытие (там тесты по `mindmap`, `text`, `frame`, `zoom`, `selection` и другим сценариям).

## Цель рефактора

Сделать структуру e2e-тестов нейтральной и предметной:
- убрать зависимость именования от устаревшего объекта `image-object2`;
- сгруппировать тесты по функциональным областям;
- сохранить текущий запуск `playwright test` и обратную совместимость по качеству прогона.

## Предлагаемая структура

- `tests/e2e/mindmap/`
- `tests/e2e/selection/`
- `tests/e2e/tools/`
- `tests/e2e/grid/`
- `tests/e2e/panels/`
- `tests/e2e/files/`

> Допускается другая итоговая группировка, если она последовательна и документирована.

## Ограничения

- Не менять поведение продукта в рамках этого рефактора.
- Не менять тестовую логику без необходимости (только пути/импорт/общие хелперы при переносе).
- После каждого шага держать тесты запускаемыми.

## Критерии готовности

- Нет e2e-файлов в `tests/image-object2/`.
- Все e2e находятся в новой нейтральной структуре.
- `npm run test:e2e` проходит.
- Обновлена документация по структуре e2e (минимум этот файл).
